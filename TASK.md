# Slot Math Sandbox

Веб-страница, которая лежит в публичном репозитории и запускается через GitHub Pages.
Это прототип псевдослота для исследования математики распределений.

**Репозиторий:** `slot-math-sandbox`

**Стек:** один HTML файл + vanilla JS. Никаких фреймворков, никакого npm. Canvas для всех графиков.

---

## Конвенция: проценты только на границе UI

**Проценты допускаются только в двух местах:**
- Поля ввода в Config (пользователь вводит 0–100)
- Отображение пользователю (строки вида `"96.00%"`)

**Все расчёты и промежуточное хранение — в десятичных дробях (0–1):**
- `cfg.rows[i].freq` хранится как `0.05` (не `5`)
- При чтении из `<input>`: делить на 100 → `(parseFloat(val) || 0) / 100`
- При записи в `<input>`: умножать на 100 → `row.freq * 100`
- При отображении строкой: `(row.freq * 100).toFixed(2) + '%'`
- `computeLinePMF` и воркер получают `freq` в десятичных: `rnd = Math.random()`, `cum += row.freq`
- RTP в процентах считается как `freq * payout * lines * 100`

## Архитектура

Вкладки переключаются без перезагрузки страницы (SPA-стиль).
При переключении вкладки — симуляция останавливается.
Симулятор живёт в **Web Worker** — не блокирует UI.
Запуск и остановка симуляции — одна кнопка (Toggle).

---

## Модель слота

Слот описывается таблицей распределения и количеством линий.

**Ставка за спин = 1** (фиксировано).

Каждый спин: для каждой линии независимо выбираем выигрыш по весам из таблицы. Суммируем по всем линиям.

**Вклад строки в RTP = частота × выплата × количество линий / 100**

## Вкладки

### 1. Config

Таблица распределения:

| Частота (%) | Выплата (x) | Вклад в RTP (%) |
|-------------|-------------|-----------------|
| input       | input       | авто            |
| ...         | ...         | ...             |
| N (авто)    | 0x          | 0               |

- Последняя строка — нулевой выигрыш — рассчитывается автоматически: `N = 100 - сумма частот`
- Если N < 0 — подсветить красным
- Строки можно добавлять и удалять
- Сортировка по частоте (по убыванию)

**Dropdown:** количество линий (1, 5, 10, 20, 50)

**Итоговый RTP:** сумма вкладов, пересчитывается в реалтайме

**Кнопка "Fit to 96%":** пропорционально масштабирует частоты до RTP = 96%. Если невозможно — предупреждение.

---

### 2. Distribution

Аналитическая вкладка — считается мгновенно из конфига, без симуляции.

**График 1: Win Frequency**
- Ось X: размер выигрыша за спин (в единицах ставки), шаг 10
- Ось Y: частота (%)
- Вертикальные линии P10, P50, P90

**График 2: RTP Distribution**
- Ось X: размер выигрыша за спин, шаг 10
- Ось Y: вклад в RTP (частота × выигрыш)
- Вертикальные линии P10, P50, P90

---

### 3. Session Outcome

Симулирует сессии и показывает распределение итогов.

**Поле:** количество спинов в сессии (по умолчанию 200)
**Кнопка:** Simulate / Stop (одна кнопка)

Во время симуляции конфиг заблокирован.

**Гистограмма:**
- Ось X: итог сессии в единицах ставки. От `-(количество спинов)` до максимально наблюдаемого, но не шире экрана. Всё что правее границы — в бакет "max+". Шаг 10.
- Обновляется в реалтайме по мере накопления статистики
- Вертикальные линии P10, P50, P90

---

### 4. ROE — Rate of Exhaustion

Симулирует игроков до банкротства.

**Поле:** стартовый банк в бетах (по умолчанию 100)
**Константа T = 20** (внутренняя переменная) — лимит спинов = bank × T
**Кнопка:** Simulate / Stop

Каждый игрок играет пока банк ≤ 0 или не достигнет лимита.

**Выводим:**
- Среднее спинов до банкротства
- Медиана спинов до банкротства
- Гистограмма распределения спинов, шаг 10, вертикальные линии P10, P50, P90

---

### 5. Bankroll

Два режима — переключатель между ними:

**Режим А: Simulate**
- Поле: количество игроков
- Поле: банк в бетах
- Лимит спинов = bank × T (та же константа T = 20)
- Кнопка Simulate / Stop
- Симулирует из текущего конфига

**Режим Б: Load JSON**
- Кнопка загрузки файла
- Формат: массив массивов — каждый вложенный массив это кривая баланса одного игрока:
```json
[
  [100, 98, 95, 110, 80, 0],
  [100, 102, 99, 88, 45, 0]
]
```

**График:** все кривые балансов на одном canvas.

> Папка `temp/` в репозитории содержит пример файла и образец реализации графика. Реализовать в том же стиле.
